# 知識メモ: DuckDB-WASM と統計計算

このプロジェクトで出てきた「初めて見るかもしれない概念」の解説なのだ。

## DuckDB-WASM とは
DuckDB（列指向の分析用DB）を **WebAssembly** にコンパイルし、ブラウザの中で動かせるようにしたもの。
- サーバもインストールも不要。ブラウザのタブの中に「小さなDB」が立ち上がるイメージ。
- CSV/Parquet を `fetch` してきて `registerFileText` で仮想ファイル登録 → `read_csv` でテーブル化。
- SQL の集計（`avg`, `stddev_samp`, `FILTER`句）がそのまま使える。
- エンジン本体(wasm)と Worker は CDN(jsDelivr)から取得する構成にしている（`src/db.ts`）。

なぜ採用したか: 「ローカルでDB管理」という要望に、サーバを立てずブラウザ内DBで応える形が
一番シンプルで、静的ホスティング（GitHub Pages 等）にそのまま載るから。

## Web Worker と blob URL
DuckDB は重い処理を **別スレッド(Web Worker)** で動かす。CDN上の worker スクリプトを
`importScripts` する小さなラッパを `Blob` にして `URL.createObjectURL` で起動している。
→ メインスレッド(UI)を固まらせないための定番テク。

## 統計の用語
- **z スコア** = (値 − 平均) / 標準偏差。「平均から標準偏差いくつ分離れているか」。
- **偏差値** = 50 + 10 × z。z=0(平均)で50、z=1で60、z=2で70。日本の受験でおなじみ。
- **正規分布 CDF (Φ)** = ある値以下になる確率。erf(誤差関数)の近似で計算している。
- **中間順位法**: 上位%を「厳密に良い件数 + 同値の半分」÷全件 で出す方法。
  丸めで同じ値が大量に出ても順位が偏らない。ties(同値)を公平に扱う標準的なやり方。

## Box-Muller 変換
一様乱数(0〜1)を2つ使って、正規分布に従う乱数を作る公式。
`sqrt(-2 ln u) * cos(2π v)`。公表されている平均/SDから「それっぽいサンプル群」を
人工的に作るために使っている（`scripts/generate-data.ts`）。

## seed 付き PRNG (mulberry32)
乱数に「種(seed)」を与えて、**毎回同じ結果**を出せるようにした乱数生成器。
CSV を再生成しても中身が変わらない＝再現可能でレビューしやすい。
